Global Variables CSV Parameters
===============================

These notes describe the compact dataset config CSVs:

- configs/physionet-global-variables.csv
- configs/mimic-global-variables.csv

`src/dataset_config.py` is the executable source of truth for the schema. This
document mirrors the current compact contract and explains how scripts use it.

CSV format
----------

- The header must exactly match `COMPACT_CONFIG_FIELDS` in
  `src/dataset_config.py`, including order.
- Each config file currently has one dataset row.
- Scalars are read directly from that row.
- Lists are JSON arrays in one cell, for example `["Age","Gender"]`.
- Booleans use values such as `True` or `False`.
- CLI arguments override CSV values when a script supports both.

Current compact field contract
------------------------------

- PREFERRED_ENV_NAME: preferred conda/runtime environment name for provenance.
- SEED: integer seed used by splits, models, and permutations.
- DATASET_NAME: human-readable dataset name.
- ID_COL: canonical row/patient identifier column, usually `ts_id`.
- ALT_ID_COL: optional alternate id accepted by some dataset-neutral scripts.
- OUTCOME_COL: dataframe outcome column, for example `in_hospital_mortality`.
- GRAPH_OUTCOME_NODE: dataset-specific outcome node in the causal DAG.
- TREATMENTS: JSON list of latent treatment graph-node names to evaluate.
- LATENT_ORDER: JSON list defining expected latent-state order.
- BACKGROUND_FEATURE_COLUMNS: JSON list of baseline covariates loaded from `ts`.
- EFFECT_MODIFIER_COLUMNS: JSON list of CATE effect modifier columns.
- MODEL_TYPE: estimator family such as `CausalForest` or `LinearDML`.
- TRIALS: integer number of permutation trials used by `main.py` and
  `src/permutations_test.py`.
- DOWN_SAMPLE: whether supported estimators down-sample the majority outcome
  class.
- USE_EXPANDED_SAFE_CONFOUNDERS: whether DAG logic uses the expanded safe
  confounder filter.
- SAVE_CONTOUR_PLOT: whether saved CATE analysis writes contour plots when
  supported by the analysis path.
- MATCH_WITH_REPLACEMENT: whether matching may reuse controls.
- REQUIRE_BINARY_CONF: whether matching rejects non-binary confounders instead
  of median-binarizing them.

Path and artifact parameters
----------------------------

The current compact CSVs intentionally do not store input/output artifact paths.
Paths are supplied by CLI flags or by script-local legacy defaults. This keeps
the dataset config focused on dataset/run semantics and avoids many duplicate
per-script path columns.

Common path flags are documented in `SCRIPTS.md`. Important examples:

- `--latent-tags-path`: latent-tag CSV input.
- `--dataset-pkl-path`: processed `[ts, oc, ts_ids]` pickle input.
- `--graph-pkl-path`: causal graph pickle input or output, depending on script.
- `--output-dir`: run output directory.
- `--results-dir`: saved CATE artifact root for analysis.
- `--experiment-dir`: permutation experiment output root.

`--physionet-pkl-path` remains a backward-compatible alias for
`--dataset-pkl-path` in dataset-neutral scripts that still expose the old flag.

Duplicate-minimization rule
---------------------------

Do not add script-specific duplicate columns unless the code explicitly needs a
durable dataset-level default. Prefer CLI flags for per-run paths and keep the
CSV compact.

If a new CSV field is needed, update all of these together:

- `COMPACT_CONFIG_FIELDS` and typed key sets in `src/dataset_config.py`
- script config contracts in `src/dataset_config.py`
- both dataset CSV headers and values
- this document
- `scripts/validate_global_variables_config.py` expectations, if relevant

Dataset and causal parameters
-----------------------------

The dataframe outcome and graph outcome are deliberately separate:

- `OUTCOME_COL` names the outcome column after loading data.
- `GRAPH_OUTCOME_NODE` names the corresponding outcome node inside the DAG.

Treatment and latent lists use graph-node names in the config. Scripts map those
to dataframe latent-tag columns where needed.

Estimation parameters
---------------------

`MODEL_TYPE`, `DOWN_SAMPLE`, `USE_EXPANDED_SAFE_CONFOUNDERS`, and `SEED` provide
run defaults. CLI values still win when exposed by the script.

Tagging and preprocessing parameters
------------------------------------

`LATENT_ORDER` is shared by taggers and plotting. Tagging thresholds and
preprocessing paths are not part of the current compact CSV contract; pass them
through the relevant script flags, such as `--thresholds-path`,
`--raw-data-path`, and `--output-path`.

Validation
----------

Use these checks after changing either config CSV or this document:

- `python3 scripts/validate_global_variables_config.py`
- `./scripts/validate_global_variables_config.sh`

The shell check also runs active scripts with `--validate-config-only` in the
configured conda environment.

Warning
-------

Do not edit scientific DAG edges or latent decision-rule logic through these
CSV files unless the code explicitly supports it. The compact configs set
dataset identifiers, column/node names, feature lists, and run defaults; they
are not a replacement for the causal model code.
